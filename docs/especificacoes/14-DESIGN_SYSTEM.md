# 14-DESIGN_SYSTEM.md

# Design System

## Objetivo

Definir o padrão visual oficial do CarCenter PRO Finance para garantir
consistência em todas as telas.

------------------------------------------------------------------------

# Identidade

Estilo: Corporativo, limpo e moderno.

Prioridades:

-   Clareza
-   Rapidez
-   Legibilidade
-   Pouca poluição visual

------------------------------------------------------------------------

# Idioma

Toda a interface deverá estar em Português (Brasil).

------------------------------------------------------------------------

# Cores

## Primária

Azul institucional.

Uso:

-   Botões principais
-   Links
-   Destaques

## Neutras

Branco Cinza claro Cinza médio Cinza escuro

## Feedback

Sucesso: Verde

Aviso: Amarelo

Erro: Vermelho

Informação: Azul

------------------------------------------------------------------------

# Tipografia

Fonte recomendada:

Inter

Hierarquia:

-   Título
-   Subtítulo
-   Texto
-   Legenda

------------------------------------------------------------------------

# Espaçamento

Utilizar grid de 8px.

Margens consistentes entre cards e componentes.

------------------------------------------------------------------------

# Componentes

Obrigatórios:

-   Button
-   Card
-   Badge
-   Input
-   Select
-   Date Picker
-   Table
-   Modal
-   Toast
-   Alert Dialog
-   Tooltip
-   Tabs

Todos devem ser reutilizáveis.

------------------------------------------------------------------------

# Botões

## Primário

Salvar

Registrar

Entrar

Confirmar

## Secundário

Cancelar

Voltar

Limpar

## Crítico

Excluir

Remover

------------------------------------------------------------------------

# Cards

Todos os cards devem conter:

-   Título
-   Conteúdo
-   Valor principal (quando aplicável)

Opcional:

-   Ícone
-   Indicador de tendência

------------------------------------------------------------------------

# Tabelas

Todas as tabelas devem permitir:

-   Ordenação
-   Pesquisa
-   Paginação
-   Exportação (quando aplicável)

------------------------------------------------------------------------

# Ícones

Utilizar ícones monocromáticos.

Sem ilustrações decorativas.

------------------------------------------------------------------------

# Layout

Cabeçalho fixo.

Menu lateral fixo.

Conteúdo central.

Rodapé discreto contendo:

-   Usuário logado
-   Versão do sistema

------------------------------------------------------------------------

# Responsividade

Prioridade:

1.  Desktop
2.  Notebook
3.  Tablet

Celular não faz parte da versão 1.0.

------------------------------------------------------------------------

# Feedback ao Usuário

Toda ação deverá apresentar retorno visual.

Exemplos:

Sucesso: Operação realizada com sucesso.

Erro: Não foi possível concluir a operação.

Processamento: Exibir indicador de carregamento.

------------------------------------------------------------------------

# Acessibilidade

-   Contraste adequado.
-   Navegação por teclado.
-   Labels em todos os campos.
-   Mensagens claras.

------------------------------------------------------------------------

# Critérios de Aceite

Todas as telas deverão utilizar exclusivamente os componentes e padrões
definidos neste documento.
