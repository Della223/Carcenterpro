# 18-ACCEPTANCE_CRITERIA.md

# Critérios de Aceite

## Objetivo

Definir os critérios mínimos que cada módulo deverá atender para ser
considerado concluído e apto para uso em produção.

------------------------------------------------------------------------

# Critérios Gerais

Todos os módulos deverão:

-   Estar totalmente em Português (Brasil).
-   Exibir mensagens claras de sucesso e erro.
-   Registrar auditoria das operações.
-   Respeitar as regras de negócio documentadas.
-   Integrar automaticamente com os módulos dependentes.
-   Possuir interface responsiva para desktop e notebook.
-   Validar dados no frontend e backend.

------------------------------------------------------------------------

# HOME

Aceite quando:

-   Exibir saudação dinâmica.
-   Exibir usuário logado.
-   Exibir competência atual.
-   Exibir cards financeiros atualizados.
-   Exibir painel de marketing.
-   Exibir assistente gerencial.
-   Navegação funcional para todos os módulos.

------------------------------------------------------------------------

# Comercial

Aceite quando:

-   Permitir cadastrar receitas.
-   Validar campos obrigatórios.
-   Calcular ticket médio.
-   Atualizar HOME, Dashboard e DRE automaticamente.
-   Permitir pesquisa, filtros e exportação.

------------------------------------------------------------------------

# Despesas

Aceite quando:

-   Permitir cadastro de despesas.
-   Gerar parcelas automaticamente.
-   Registrar pagamentos.
-   Atualizar saldo remanescente.
-   Atualizar DRE pela competência.
-   Integrar com Dashboard e Relatórios.

------------------------------------------------------------------------

# Dashboard

Aceite quando:

-   Exibir indicadores em tempo real.
-   Aplicar filtros globais.
-   Exibir gráficos obrigatórios.
-   Não permitir edição.

------------------------------------------------------------------------

# DRE

Aceite quando:

-   Gerar automaticamente.
-   Não permitir edição manual.
-   Considerar apenas receitas registradas e parcelas pagas na
    competência.
-   Exibir comparativo Orçado x Realizado.

------------------------------------------------------------------------

# Orçamentos

Aceite quando:

-   Permitir orçamento anual.
-   Comparar realizado x orçado.
-   Emitir alertas de execução.

------------------------------------------------------------------------

# Relatórios

Aceite quando:

-   Gerar relatórios conforme filtros.
-   Exportar PDF, Excel e CSV.
-   Permitir impressão formatada.

------------------------------------------------------------------------

# Configurações

Aceite quando:

-   Permitir manutenção de cadastros.
-   Refletir alterações imediatamente.
-   Manter auditoria.
-   Não permitir exclusão de registros utilizados.

------------------------------------------------------------------------

# Marketing

Aceite quando:

-   Exibir lembrete de Story (segunda a sexta).
-   Exibir lembrete de Feed (terça e quinta).
-   Registrar automaticamente usuário, data e hora ao marcar publicação.

------------------------------------------------------------------------

# Sistema

Aceite quando:

-   Todos os módulos integrarem corretamente.
-   Não houver duplicidade de dados.
-   Todas as regras de negócio forem respeitadas.
-   Tempo médio de carregamento inferior a 3 segundos nas telas
    principais.

------------------------------------------------------------------------

# Critério Final de Aceite

O CarCenter PRO Finance será considerado pronto para a versão 1.0 quando
todos os critérios descritos neste documento forem atendidos
integralmente.
