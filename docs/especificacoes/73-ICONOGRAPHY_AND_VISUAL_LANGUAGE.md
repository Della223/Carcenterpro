# 73-ICONOGRAPHY_AND_VISUAL_LANGUAGE.md

# Iconografia e Linguagem Visual

## Objetivo

Padronizar a linguagem visual do CarCenter PRO Finance, garantindo
consistência na utilização de ícones, indicadores, badges e elementos
gráficos.

------------------------------------------------------------------------

# Biblioteca Oficial

Biblioteca recomendada:

-   Lucide React

Regras:

-   Mesmo estilo em toda a aplicação.
-   Ícones vetoriais (SVG).
-   Nunca misturar bibliotecas diferentes.

------------------------------------------------------------------------

# Tamanhos

-   Pequeno: 16px
-   Padrão: 20px
-   Grande: 24px
-   Destaque: 32px

Utilizar sempre múltiplos da escala do Design System.

------------------------------------------------------------------------

# Cores

Neutro: - Ícones informativos.

Primária: - Ações principais.

Verde: - Sucesso.

Amarelo: - Aviso.

Vermelho: - Erro.

Azul: - Informação.

As cores devem utilizar Design Tokens.

------------------------------------------------------------------------

# Ícones por Módulo

HOME - house - bell - chart-column

Comercial - dollar-sign - receipt

Despesas - wallet - credit-card

Dashboard - chart-line - chart-pie

DRE - file-text - calculator

Orçamentos - target

Relatórios - file-bar-chart

Configurações - settings

Usuário - user

Logout - log-out

------------------------------------------------------------------------

# Indicadores de Status

Sucesso: ✔

Aviso: ⚠

Erro: ✖

Informação: ℹ

Os ícones devem ser acompanhados por texto quando necessários para
acessibilidade.

------------------------------------------------------------------------

# Badges

Tipos:

-   Success
-   Warning
-   Error
-   Info
-   Neutral

Uso:

-   Status
-   Situação
-   Competência
-   Perfil

------------------------------------------------------------------------

# Chips

Utilização:

-   Filtros ativos
-   Categorias
-   Centros de custo

Devem permitir remoção rápida quando aplicável.

------------------------------------------------------------------------

# Estados Visuais

Loading: Skeleton.

Empty: Ícone ilustrativo + mensagem.

Error: Ícone de erro + ação "Tentar novamente".

------------------------------------------------------------------------

# Acessibilidade

-   Todos os ícones decorativos devem ser marcados como aria-hidden.
-   Ícones com função devem possuir rótulo acessível.
-   Contraste conforme WCAG AA.

------------------------------------------------------------------------

# Convenções

-   Nunca utilizar apenas cor para transmitir significado.
-   Sempre combinar cor + ícone + texto quando relevante.
-   Evitar excesso de ícones na mesma tela.

------------------------------------------------------------------------

# Critérios de Aceite

-   Biblioteca única de ícones.
-   Consistência visual em todos os módulos.
-   Compatibilidade com o Design System.
-   Acessibilidade atendida.
