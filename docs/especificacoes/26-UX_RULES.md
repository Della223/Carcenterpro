# 26-UX_RULES.md

# Regras de UX

## Objetivo

Padronizar a experiência do usuário em todos os módulos do CarCenter PRO
Finance.

------------------------------------------------------------------------

# Princípios

-   Interface simples.
-   Poucos cliques.
-   Feedback imediato.
-   Consistência visual.
-   Prioridade para produtividade.

------------------------------------------------------------------------

# Navegação

-   Menu lateral sempre visível.
-   Cabeçalho fixo.
-   Nunca abrir novas abas para módulos internos.
-   Sempre destacar o módulo ativo.

------------------------------------------------------------------------

# Formulários

-   Foco automático no primeiro campo.
-   Navegação por TAB.
-   Campos obrigatórios identificados.
-   Salvar e Cancelar sempre no rodapé do formulário.
-   Não perder dados digitados em caso de erro de validação.

------------------------------------------------------------------------

# Pesquisa

-   Campo de pesquisa sempre visível quando aplicável.
-   Busca em tempo real após pequena pausa de digitação.
-   Limpar pesquisa com um clique.

------------------------------------------------------------------------

# Filtros

-   Manter filtros aplicados durante a navegação.
-   Botões:
    -   Filtrar
    -   Limpar
-   Exibir quantidade de registros encontrados.

------------------------------------------------------------------------

# Tabelas

-   Cabeçalho fixo.
-   Ordenação por coluna.
-   Paginação.
-   Seleção de quantidade de registros por página.
-   Exportação quando disponível.

------------------------------------------------------------------------

# Confirmações

Solicitar confirmação apenas para ações destrutivas:

-   Excluir
-   Cancelar operações irreversíveis

Nunca solicitar confirmação para salvar.

------------------------------------------------------------------------

# Feedback

Sucesso: Toast verde.

Erro: Toast vermelho.

Aviso: Toast amarelo.

Informação: Toast azul.

------------------------------------------------------------------------

# Atalhos de Teclado

Ctrl + S → Salvar

Esc → Fechar modal

Enter → Confirmar formulário quando apropriado

------------------------------------------------------------------------

# Carregamento

-   Skeleton para tabelas, cards e gráficos.
-   Nunca exibir tela em branco durante carregamento.

------------------------------------------------------------------------

# Empty State

Quando não houver dados:

-   Explicar a situação.
-   Oferecer ação para criar o primeiro registro, quando aplicável.

------------------------------------------------------------------------

# Responsividade

Prioridade:

1.  Desktop
2.  Notebook
3.  Tablet

Celular fora do escopo da versão 1.0.

------------------------------------------------------------------------

# Acessibilidade

-   Labels em todos os campos.
-   Contraste adequado.
-   Navegação por teclado.
-   Mensagens compreensíveis.

------------------------------------------------------------------------

# Critérios de Aceite

Todas as telas devem seguir estas regras de usabilidade, garantindo uma
experiência consistente e previsível ao usuário.
